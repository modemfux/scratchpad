# Хранение и передача битов и байтов

[**Назад**](/README.md)

Число `170` можно представить в виде двоичной записи `0b10101010`, где:

- `0b` - обозначение двоичной записи (`0b` - двоичная, `0o` - восмеричная, `0x` - шестнадцатиричная).

| Value | Postion | Bit multiplier |       Description     |
|:-----:|:-------:|:--------------:|-----------------------|
|  `1`  |   `7`   |     `128`      | Most Significant Bit  |
|  `0`  |   `6`   |      `64`      | -                     |
|  `1`  |   `5`   |      `32`      | -                     |
|  `0`  |   `4`   |      `16`      | -                     |
|  `1`  |   `3`   |       `8`      | -                     |
|  `0`  |   `2`   |       `4`      | -                     |
|  `1`  |   `1`   |       `2`      | -                     |
|  `0`  |   `0`   |       `1`      | Least Significant Bit |

Передача битов из байта также может быть реализована двумя способами:

**MSB-first (Most Significant Bit first)** - сначала передается старший бит в байте.

Разберем на примере: Алиса передает байт `0xAA` (`0b10101010`) Бобу. В случае с MSB-first передача будет выглядеть так:

1. Исходное состояние. Aлиса: `10101010`, Боб: `xxxxxxxx`.
2. Был передан `1`. Aлиса: `x0101010`, Боб: `1xxxxxxx`.
3. Был передан `0`. Aлиса: `xx101010`, Боб: `10xxxxxx`.
4. Был передан `1`. Aлиса: `xxx01010`, Боб: `101xxxxx`.
5. Был передан `0`. Aлиса: `xxxx1010`, Боб: `1010xxxx`.
6. Был передан `1`. Aлиса: `xxxxx010`, Боб: `10101xxx`.
7. Был передан `0`. Aлиса: `xxxxxx10`, Боб: `101010xx`.
8. Был передан `1`. Aлиса: `xxxxxxx0`, Боб: `1010101x`.
9. Был передан `0`. Aлиса: `xxxxxxxx`, Боб: `10101010`.

**LSB-first (Least Significant Bit first)** - сначала передается младший бит в байте.

Разберем на примере: Алиса передает байт `0xAA` (`0b10101010`) Бобу. В случае с LSB-first передача будет выглядеть так:

1. Исходное состояние. Aлиса: `10101010`, Боб: `xxxxxxxx`.
2. Aлиса: `1010101x`, `0` -> Боб: `xxxxxxx0`.
3. Aлиса: `101010xx`, `1` -> Боб: `xxxxxx10`.
4. Aлиса: `10101xxx`, `0` -> Боб: `xxxxx010`.
5. Aлиса: `1010xxxx`, `1` -> Боб: `xxxx1010`.
6. Aлиса: `101xxxxx`, `0` -> Боб: `xxx01010`.
7. Aлиса: `10xxxxxx`, `1` -> Боб: `xx101010`.
8. Aлиса: `1xxxxxxx`, `0` -> Боб: `x0101010`.
9. Aлиса: `xxxxxxxx`, `1` -> Боб: `10101010`.

Храниться и передаваться два этих байта могут в двух видах:

- **Big Endian** - старший байт (most significant byte, MSB) хранится по младшему адресу памяти.
- **Little Endian** - младший байт (least significant byte, LSB) хранится по младшему адресу памяти.

На примерах:

2-байтовое число: `51 966` (`0xCAFE`).

**Big-endian:**

|  Address | Data Value |
|:--------:|:----------:|
| `0x1000` |   `0xCA`   |
| `0x1001` |   `0xFE`   |

Линейная запись: `0xCA`, `0xFE`.

**Little-endian:**

|  Address | Data Value |
|:--------:|:----------:|
| `0x1000` |   `0xFE`   |
| `0x1001` |   `0xCA`   |

Линейная запись: `0xFE`, `0xCA`.

4-байтовое число: `3 735 931 646` (`0xDEADCAFE`).

**Big-endian:**

|  Address | Data Value |
|:--------:|:----------:|
| `0x1000` |   `0xDE`   |
| `0x1001` |   `0xAD`   |
| `0x1002` |   `0xCA`   |
| `0x1003` |   `0xFE`   |

Линейная запись: `0xDE`, `0xAD` ,`0xCA`, `0xFE`.

**Little-endian:**

|  Address | Data Value |
|:--------:|:----------:|
| `0x1000` |   `0xFE`   |
| `0x1001` |   `0xCA`   |
| `0x1002` |   `0xAD`   |
| `0x1003` |   `0xDE`   |

Линейная запись: `0xFE`, `0xCA` ,`0xAD`, `0xDE`.
